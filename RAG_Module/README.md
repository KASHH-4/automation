# AI Proposal Intelligence System — RAG Module

> **Role in the project:** Knowledge Base & Retrieval Engineering
> **Maintainer:** RAG Engineer
> **Integrates with:** Preprocessing Pipeline (notebooks 01–04) and Proposal Generation System

---

## Overview

This module is the **retrieval backbone** of the AI Proposal Intelligence System.

Given a natural-language query (e.g., *"stainless steel pipe quote for 150 kg, 10-day delivery"*), it searches a FAISS vector index of cleaned historical quotation data and returns the most semantically similar past quotes — ranked by relevance — ready for downstream LLM use.

```
User Query
    │
    ▼
retrieve_similar_quotes(query, k=3)
    │
    ├── Embed query  (Sentence Transformer: all-MiniLM-L6-v2)
    ├── Search FAISS index  (ANN similarity)
    └── Return top-K results with scores + metadata
```

---

## Full Project Pipeline

The RAG module sits **at the end** of a preprocessing pipeline. Run steps in order:

```
Step 1 — 01_excel_ingestion.ipynb
          Reads raw Excel supplier sheets → quotations_raw.json
          ↓

Step 2 — 02_pdf_ingestion.ipynb
          Reads raw quotation PDFs → extracts structured fields
          ↓

Step 3 — 03_market_api.py
          Fetches live exchange rates (USD/EUR/GBP → INR) → exchange_rates.json
          ↓

Step 4 — 04_clean_validate.ipynb
          Cleans, validates, and merges all data → clean_data.json
          ↓

Step 5 — Copy clean_data.json to rag_module/data/
          cp /path/to/project/output/clean_data.json data/
          (or set the RAG_JSON_PATH environment variable)
          ↓

Step 6 — python build_index.py
          Loads clean_data.json → splits → embeds → builds FAISS index
          ↓

Step 7 — python retriever.py
          Queries the index and returns enriched quotation results
```

> **Important:** `build_index.py` reads `data/clean_data.json` — it does **not** read raw PDFs.
> The raw PDFs are processed by notebooks 01–04, not by this module.

## Configuration (`config.py`)

Single source of truth for all settings. Change any value here — no other file needs editing.

| Constant | Default | Description |
|---|---|---|
| `DATA_SOURCE_TYPE` | `"json"` | `"json"` reads clean_data.json; `"pdf"` reads PDFs |
| `JSON_PATH` | `data/clean_data.json` | Path to cleaned dataset (overridable via `RAG_JSON_PATH` env var) |
| `JSON_KEY` | `"quotations"` | Key inside clean_data.json that holds the records list |
| `EMBEDDING_MODEL_NAME` | `all-MiniLM-L6-v2` | 384-dim Sentence Transformer |
| `CHUNK_SIZE` | `800` | Max characters per text chunk |
| `CHUNK_OVERLAP` | `150` | Overlap characters between chunks |
| `DEFAULT_TOP_K` | `3` | Results returned per query |

## Module Details

### `loader.py`

- **Primary path:** reads `clean_data.json` (nested dict with `"quotations"` key)
- Converts each quotation record into a `LangChain Document` with structured text and metadata
- **Legacy path:** `DATA_SOURCE_TYPE="pdf"` falls back to reading PDFs from `PDF_DIR`
- Skips malformed records without crashing
- All errors are caught and logged — the pipeline continues

### `splitter.py`

- Uses `RecursiveCharacterTextSplitter`
- Splits on paragraph → line → sentence → word boundaries
- Each chunk inherits parent metadata (`source_file`, `quotation_id`, `customer`, etc.)

### `embeddings.py`

- Wraps `sentence-transformers/all-MiniLM-L6-v2` via `HuggingFaceEmbeddings`
- `normalize_embeddings=True` → unit vectors → cosine similarity
- Model cached locally after first download (~90 MB)
- **Must use the same model** for both indexing and querying

### `build_index.py`

- Orchestrates the full pipeline in 4 steps
- Saves two files: `vector_store/index.faiss` + `vector_store/index.pkl`
- Re-run whenever `clean_data.json` is updated

### `retriever.py`

**Public function:**

```python
def retrieve_similar_quotes(query: str, k: int = 3) -> List[dict]:
```

**Return format:**
```python
[
    {
        "quotation_id"    : "QTN-2026-001",
        "customer"        : "ABC Manufacturing Pvt. Ltd.",
        "supplier"        : "Tata Steel",
        "currency"        : "INR",
        "source_file"     : "Quotation 1 (3).pdf",
        "file_name"       : "Quotation 1 (3).pdf",   # backward compat
        "content"         : "Quotation ID: QTN-2026-001\nCustomer: ...",
        "similarity_score": 0.9241,
        "metadata"        : { ...full metadata dict... }
    },
    ...
]
```

**Score formula:** `score = 1 / (1 + L2_distance)` — converts FAISS L2 distance to a 0→1 score.

**Performance:** The FAISS index is loaded once and cached in module memory.

## Integration Guide (for teammates)

Your proposal generation module only needs to import one function:

```python
# In proposal_generator.py
from rag_module.retriever import retrieve_similar_quotes

def generate_proposal(client_request: str) -> str:
    # Step 1: Retrieve relevant historical quotes
    similar_quotes = retrieve_similar_quotes(client_request, k=3)

    # Step 2: Build context for your LLM prompt
    context_parts = []
    for q in similar_quotes:
        context_parts.append(
            f"[{q['quotation_id']}] Customer: {q['customer']} | "
            f"Supplier: {q['supplier']} | Score: {q['similarity_score']}\n"
            f"{q['content']}"
        )
    context = "\n\n".join(context_parts)

    # Step 3: Pass context + request to your LLM
    prompt = (
        f"Based on these past quotations:\n{context}\n\n"
        f"Generate a proposal for: {client_request}"
    )
    # ... your LLM call here
```

**Prerequisites before calling `retrieve_similar_quotes()`:**
1. `clean_data.json` must be present at `data/clean_data.json` (or `RAG_JSON_PATH` set)
2. `python build_index.py` must have been run at least once
3. `vector_store/index.faiss` and `vector_store/index.pkl` must exist

## Switching Back to PDF Mode (Legacy)

To fall back to the original PDF-based pipeline without any code changes:

1. Set `DATA_SOURCE_TYPE = "pdf"` in `config.py`
2. Copy PDFs into `data/` (sample PDFs are in `sample_data/`)
3. Run `python build_index.py`

## Upgrading the Embedding Model

Change only one line in `config.py`:

```python
# EMBEDDING_MODEL_NAME = "all-MiniLM-L6-v2"   # 384-dim, fast (default)
EMBEDDING_MODEL_NAME = "all-mpnet-base-v2"      # 768-dim, more accurate
```

Then **rebuild the index**: `python build_index.py`

> ⚠️ You cannot mix models between index building and retrieval — always rebuild after changing the model.
